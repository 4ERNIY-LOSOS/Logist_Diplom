# Руководство по настройке рабочего окружения

Этот документ описывает все шаги для установки и настройки программного обеспечения, необходимого для разработки нашего проекта на стеке **NestJS (Backend) + React/Tauri (Frontend) + PostgreSQL (База данных)**.

---

### Шаг 1: Основные инструменты

Это базовое ПО, которое понадобится в любом случае.

1.  **Система контроля версий Git:**
    *   **Что это:** Инструмент для отслеживания версий вашего кода.
    *   **Установка:** Скачайте и установите с [официального сайта Git](https://git-scm.com/downloads).

2.  **Редактор кода Visual Studio Code (VS Code):**
    *   **Что это:** Наш основной редактор для написания кода.
    *   **Установка:** Скачайте и установите с [официального сайта VS Code](https://code.visualstudio.com/).

---

### Шаг 2: Настройка Backend (Сервер)

1.  **Node.js и NPM:**
    *   **Что это:** Среда выполнения JavaScript на сервере и менеджер пакетов.
    *   **Установка:** Скачайте и установите **LTS-версию** (Long-Term Support) с [официального сайта Node.js](https://nodejs.org/). NPM установится автоматически вместе с Node.js.
    *   **Проверка:** Откройте терминал (командную строку) и выполните `node -v` и `npm -v`. Вы должны увидеть версии программ.

2.  **NestJS CLI:**
    *   **Что это:** Командная утилита для быстрого создания и управления проектом NestJS.
    *   **Установка:** В терминале выполните команду:
        ```bash
        npm install -g @nestjs/cli
        ```

---

### Шаг 3: Настройка Frontend (Клиент)

Для работы Tauri требуется несколько системных зависимостей.

1.  **Microsoft C++ Build Tools:**
    *   **Что это:** Набор инструментов от Microsoft, необходимый для компиляции некоторых зависимостей.
    *   **Установка:**
        1.  Скачайте [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/).
        2.  Запустите установщик.
        3.  Во вкладке "Рабочие нагрузки" выберите **"Разработка классических приложений на C++"** и нажмите "Установить".

2.  **Rust:**
    *   **Что это:** Язык программирования, на котором написан Tauri. Нам нужна его система сборки.
    *   **Установка:**
        1.  Перейдите на сайт [rustup.rs](https://rustup.rs/).
        2.  Скачайте `rustup-init.exe` и запустите.
        3.  В открывшемся терминале просто нажмите Enter, чтобы выбрать установку по умолчанию.

### Шаг 4: Настройка Базы Данных (PostgreSQL через Docker)

Мы не будем устанавливать PostgreSQL напрямую в систему. Мы будем использовать Docker, чтобы запустить его в изолированном контейнере. Это чистый, удобный и современный подход.

1.  **Docker Desktop:**
    *   **Что это:** Приложение для запуска контейнеров на Windows.
    *   **Установка:** Скачайте и установите с [официального сайта Docker](https://www.docker.com/products/docker-desktop/). Docker Desktop может попросить вас установить WSL2 (подсистему Windows для Linux) — соглашайтесь, это стандартная процедура.

2.  **Файл `docker-compose.yml`:**
    *   **Что это:** Файл-инструкция для Docker, описывающий, как запустить нашу базу данных.
    *   **Действие:** В корневой папке вашего проекта (`C:\Users\makar\Desktop\Diplom`) создайте файл с именем `docker-compose.yml` и скопируйте в него следующее содержимое:

    ```yml
    version: '3.8'
    services:
      postgres:
        image: postgres:14-alpine
        restart: always
        environment:
          - POSTGRES_USER=admin
          - POSTGRES_PASSWORD=admin
          - POSTGRES_DB=logistics_db
        ports:
          - '5432:5432'
        volumes:
          - postgres_data:/var/lib/postgresql/data

    volumes:
      postgres_data:
    ```

3.  **Запуск базы данных:**
    *   **Действие:** Откройте терминал в папке с файлом `docker-compose.yml` и выполните команду:
        ```bash
        docker-compose up -d
        ```
    *   Эта команда скачает образ PostgreSQL и запустит контейнер в фоновом режиме. Ваша база данных теперь работает!

4.  **GUI-клиент для базы данных:**
    *   **Что это:** Программа для удобного просмотра таблиц и данных в нашей БД.
    *   **Установка:** Скачайте и установите **DBeaver** с [официального сайта](https://dbeaver.io/download/).
    *   **Настройка подключения в DBeaver:**
        *   Создайте новое подключение к `PostgreSQL`.
        *   Хост: `localhost`
        *   Порт: `5432`
        *   База данных: `logistics_db`
        *   Пользователь: `admin`
        *   Пароль: `admin`
